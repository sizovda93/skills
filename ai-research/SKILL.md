# AI Research Skill

Задача — собрать актуальную сводку по запросу пользователя из нескольких источников и вернуть структурированный отчёт.

## Процесс

1. **Уточни скоуп** (если запрос размыт): какой таймфрейм (неделя/месяц/квартал), какая область (LLM, генерация изображений, роботика, регулирование, бизнес), какие компании интересуют.

2. **Собери данные параллельно** — запусти несколько `WebSearch` и `WebFetch` в одном сообщении:
   - WebSearch по ключевым запросам (`"<topic> site:anthropic.com"`, `"<topic> 2026"`, `"<model> release notes"`)
   - WebFetch официальных блогов если нужны детали: anthropic.com/news, openai.com/blog, deepmind.google/discover/blog, ai.meta.com/blog, mistral.ai/news, x.ai/blog
   - Для академии: arxiv.org (поиск по категории cs.AI / cs.LG), paperswithcode.com
   - Для индустрии: techcrunch.com, theverge.com/ai-artificial-intelligence, the-decoder.com

3. **Для глубокого ресерча** (>3 запросов) делегируй `Agent` с `subagent_type=general-purpose` — он параллелит веб-запросы и возвращает сводку, не засоряя основной контекст.

4. **Верни отчёт** в таком формате:

```markdown
# AI Research: <тема> (<дата>)

## TL;DR
3–5 буллетов с главным.

## Релизы моделей / продукты
- **<название>** (<компания>, <дата>) — что нового, контекст, ссылка.

## Исследования / статьи
- **<заголовок>** (<авторы>, arxiv:XXXX) — суть в одном предложении.

## Индустрия / бизнес
- Сделки, раунды, регулирование, найм.

## Что дальше / на что смотреть
Открытые вопросы, анонсированные релизы, ожидаемые события.

## Источники
Нумерованный список URL.
```

## Правила

- **Указывай даты** у каждого факта — ИИ движется быстро, "недавно" через месяц устареет.
- **Проверяй первоисточник** — если новость на TechCrunch ссылается на официальный блог, читай блог.
- **Различай факты и слухи** — помечай `[leak]`, `[rumor]`, `[unconfirmed]`.
- **Не выдумывай** — если источник недоступен, скажи "не удалось подтвердить".
- **Пиши по-русски**, если пользователь пишет по-русски; имена моделей/компаний оставляй в оригинале.
