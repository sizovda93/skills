# AI Research Skill

Задача — собрать актуальную сводку по запросу пользователя из нескольких источников (официальные блоги, академия, индустриальная пресса, YouTube, Telegram) и вернуть структурированный отчёт.

## Процесс

1. **Уточни скоуп** (если запрос размыт): таймфрейм (неделя/месяц/квартал), область (LLM, генерация изображений, агенты, роботика, регулирование, бизнес), интересующие компании или конкретные источники (например, "только Сиолошная и AI Explained").

2. **Собери данные параллельно** — запусти несколько `WebSearch` и `WebFetch` в одном сообщении. Источники см. в секции **Источники** ниже.

3. **Для глубокого ресерча** (>3 запросов) делегируй `Agent` с `subagent_type=general-purpose` — он параллелит веб-запросы и возвращает сводку, не засоряя основной контекст.

4. **Верни отчёт** в формате из секции **Шаблон отчёта** ниже.

### YouTube каналы

**Поиск свежих видео:** `WebFetch` страницу `youtube.com/@<channel>/videos` → список последних видео с заголовками и датами.

**Получение транскрипта** (если нужно содержание видео, а не только заголовок):

У пользователя две машины — **macOS** и **Windows**. На обеих установлены `yt-dlp` и `ffmpeg`:
- **macOS** — через Homebrew (`/opt/homebrew/bin/yt-dlp`)
- **Windows** — через winget (`$LOCALAPPDATA/Microsoft/WinGet/Packages/yt-dlp.yt-dlp_Microsoft.Winget.Source_8wekyb3d8bbwe/yt-dlp.exe`)

Платформу определяй по окружению (Claude Code сообщает `Platform: darwin` для мака, `win32` для винды).

Для скачивания авто-субтитров YouTube используй `Bash`:

**macOS:**
```bash
yt-dlp --write-auto-sub --sub-lang ru,en --skip-download --sub-format vtt -o "/tmp/yt-%(id)s.%(ext)s" "<URL>"
```
Файл окажется в `/tmp/yt-<videoid>.ru.vtt` или `.en.vtt`.

**Windows:**
```bash
yt-dlp --write-auto-sub --sub-lang ru,en --skip-download --sub-format vtt -o "%TEMP%/yt-%(id)s.%(ext)s" "<URL>"
```
Файл окажется в `%TEMP%/yt-<videoid>.ru.vtt`.

Затем `Read` полученный `.vtt` файл и работай с текстом. Файл содержит таймкоды + повторы строк (формат VTT с word-timing) — для саммари можно игнорировать дубли.

**Если в текущей сессии `yt-dlp` не найден** (PATH не подхватился после установки), используй полный путь для своей платформы из списка выше.

**После использования** удаляй временные `.vtt` файлы (из `/tmp/` на маке или `%TEMP%` на винде).

**Ограничения:** YouTube auto-captions на русском кривоваты (имена/термины путает) — для критичных моментов перепроверяй у первоисточника. Если auto-captions недоступны (`There are no subtitles for the requested languages`) — fallback на `WebSearch "<title> summary"`.

## Signal vs Noise: что включать в отчёт

Перед тем как добавить факт/новость в отчёт, прогоняй его через этот чеклист. Задача — дать пользователю плотную сводку сигнала, а не шума.

### Иерархия доверия к источникам

Если один и тот же факт есть в нескольких источниках, ссылайся на самый высокий уровень:

1. **Официальный блог/релиз** компании (anthropic.com, openai.com, deepmind.google, arxiv.org) — первоисточник, максимум доверия.
2. **Tier-1 пресса** (TechCrunch, The Verge, Ars Technica, The Decoder) — проверенные факты + контекст.
3. **Авторские аналитические каналы** (Сиолошная, Kantor.AI, Gonzo ML, Yannic Kilcher) — факты + разбор. Мнения автора помечай `[мнение]`.
4. **Агрегаторы/дайджесты** (ai_newz, data_secrets, AI Breakfast) — быстрые, но без глубины. Всегда ищи первоисточник.
5. **Мелкие и промо-каналы** — низкое доверие. Не используй как единственный источник.

**Правило:** если факт упоминает только источник уровня 4-5 без ссылки на 1-3 — помечай `[unverified]` или не включай.

### Cross-validation (подтверждение в нескольких источниках)

- **3+ независимых источника** с ссылкой на оригинал → факт.
- **3 канала с одинаковым текстом** → это репост одного источника, считается **как 1**. Проверяй первоисточник.
- **Только 1 источник уровня 1** (официальный блог) → факт, можно без cross-validation.
- **Только 1 источник уровня 4-5** → `[unverified]`.

### Что ВСЕГДА отфильтровывать (шум)

- ❌ **Промо-посты / реклама курсов / "топ-10 нейросетей которые..."** — даже если канал подписки.
- ❌ **Мемы и хайп без фактов** ("это меняет всё", "AI невероятен") без конкретики.
- ❌ **Общие рассуждения без новостного повода** ("как AI влияет на работу") — это колонка, не новость.
- ❌ **Новости за пределами таймфрейма** — если запрошена неделя, старое не включай, даже если интересно.
- ❌ **Кликбейт-заголовки без подтверждения в теле** — "Google удивил всех" без деталей = шум.

### Что помечать, но не выкидывать

- ⚠️ **Слухи/leaks** → `[rumor]` или `[leak]` + кто слил.
- ⚠️ **Авторские мнения** → `[мнение @канал]` — отдельно от фактов.
- ⚠️ **Маркетинг под видом прорыва** → если компания заявляет SOTA без опубликованных бенчмарков, помечай `[PR-claim, без независимой проверки]`.
- ⚠️ **Противоречия между источниками** → укажи обе версии + пометь `[противоречие]`.

### Сигналы ценности (что включать в первую очередь)

- ✅ Конкретные числа: размеры модели, бенчмарки с сопоставлением, суммы раундов, даты релизов.
- ✅ Публикации с arxiv ID + код/веса.
- ✅ Анонсы с живыми ссылками на try/API/download.
- ✅ Регуляторные решения с официальными документами.
- ✅ Заявления ключевых людей (CEO/CTO/lead researchers) на верифицированных каналах.
- ✅ Разборы от экспертов-практиков (не обзорщиков) — когда Karpathy или Сиолошная разбирают релиз, это сигнал.

### Финальная проверка перед отчётом

Перед выдачей спроси себя по каждому пункту:
1. **Дата есть?** Если нет — не включаю или ищу.
2. **Первоисточник проверен?** Если факт из TG-канала, который ссылается на блог — я открыл блог?
3. **Это факт или мнение?** Помечено соответственно?
4. **Пользователю это реально полезно?** Или просто "интересно"? Убирай "интересно без пользы".
5. **Нет ли дублей?** Одно и то же событие не должно быть в 3 секциях отчёта.

## Шаблон отчёта

```markdown
# AI Research: <тема> (<дата>)

## TL;DR
3–5 буллетов с главным.

## Релизы моделей / продукты
- **<название>** (<компания>, <дата>) — что нового, контекст, [ссылка](url).

## Исследования / статьи
- **<заголовок>** (<авторы>, arxiv:XXXX) — суть в одном предложении.

## Индустрия / бизнес
- Сделки, раунды, регулирование, найм.

## Из YouTube / Telegram
- **<канал>**: краткое содержание поста/видео + дата + ссылка.

## Что дальше / на что смотреть
Открытые вопросы, анонсированные релизы, ожидаемые события.

## Источники
Нумерованный список URL.
```

## Правила

- **Указывай даты** у каждого факта — ИИ движется быстро, "недавно" через месяц устареет.
- **Проверяй первоисточник** — если новость на TechCrunch ссылается на официальный блог, читай блог. Если YouTuber или Telegram-канал ссылается на статью/анонс — открывай оригинал.
- **Различай факты и слухи** — помечай `[leak]`, `[rumor]`, `[unconfirmed]`. Контент из YT/TG помечай как `[мнение]` если это авторский комментарий, а не факт.
- **Не выдумывай** — если источник недоступен или WebFetch вернул пустоту, явно скажи "не удалось получить данные с <источник>".
- **Пиши по-русски**, если пользователь пишет по-русски; имена моделей/компаний/каналов оставляй в оригинале.
- **Если пользователь спросил про конкретный канал** ("что в Сиолошной?") — фетчи именно его, не размазывай ресерч.
